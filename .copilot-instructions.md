# GitHub Copilot Instructions

## 🚫 DO NOT Create Documentation Files

**Default Behavior:** Do NOT create markdown (.md) files unless explicitly requested.

### What This Means

**Don't create:**
- Implementation summaries
- Quick start guides
- Fix documentation
- Change logs
- Reference manuals
- Any .md files

**Only create .md files when user explicitly says:**
- "Create documentation for this"
- "Write a README"
- "Document this feature"
- "Create a guide for..."

### How to Handle Tasks

**For features/fixes:**
1. ✅ Implement the code
2. ✅ Test it works
3. ✅ Explain in chat what you did
4. ❌ Don't create .md files

**For questions:**
1. ✅ Answer in chat
2. ✅ Provide code examples if needed
3. ❌ Don't create reference docs

### Exception Cases

**DO create documentation when:**
- User explicitly asks for it
- It's a complex API that needs endpoint docs
- Team collaboration requires it
- User says "document this"

**DON'T create documentation when:**
- Implementing features (unless asked)
- Fixing bugs (unless asked)
- Refactoring code
- Making configuration changes

### Quick Reference

| User Says | Action |
|-----------|--------|
| "Fix the dropdown" | Fix it, NO docs |
| "Add new feature" | Implement, NO docs |
| "Document this feature" | Create .md file ✅ |
| "No markdown files" | Definitely NO docs |
| (nothing about docs) | NO docs (default) |

## 💬 Communication Style

**Keep responses:**
- Concise and direct
- Focus on what was done
- Explain in chat, not files
- Provide code snippets in chat if helpful

**Don't:**
- Create summary files
- Write implementation docs
- Make quick start guides
- Document every change
